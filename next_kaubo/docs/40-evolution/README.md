## 方案评估历史

泛型系统的评估经历了多个版本的思考：

### v1（2026-02-16）

**观点**：泛型有过度设计风险，建议简化

- 担心 20 周投入过大
- 建议先做"仅内置泛型"

### v2（2026-02-16）

**观点**：泛型是类型推断的基础设施

- 认为没有泛型就没有准确类型推断
- 决定做完整版（类似 TypeScript）

### v3（2026-02-16）

**观点**：诚实修正，泛型主要解决代码复用

- 承认类型推断可以在没有泛型的情况下工作
- 泛型主要解决高阶函数和容器的类型传递
- 坚持完整路线图，但理由更清晰

详见 [assessment.md](assessment.md) 获取完整评估报告。
