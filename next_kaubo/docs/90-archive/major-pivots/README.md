# 重大决策快照

仅在发生架构级转向时创建。日常调整不需要记录。

## 什么时候创建快照

- 语法风格完全改变（如JS风格→Rust风格）
- 核心技术栈替换（如Cranelift→LLVM）
- 内存管理方式改变（如GC→Arena）
- 项目目标/用户群体改变

## 格式

每个快照文件：
```
YYYY-MM-DD-简短描述.md
```

内容：
- 触发条件（为什么转向）
- 决策内容（转向到哪里）
- 关键认知（仅此一条最重要的）
- 代码标签（git tag，方便回溯）
